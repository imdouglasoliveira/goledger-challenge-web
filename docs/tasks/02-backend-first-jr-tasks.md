## Como usar

- execute uma task por vez
- valide o criterio de pronto antes de prosseguir
- se uma task falhar, volte para a anterior e revise premissas
- registre risco residual quando nao der para corrigir de imediato

## Gates obrigatorios antes da UI

- Basic Auth funcionando
- Swagger e Scalar funcionando
- leitura de `tvShows` funcionando
- mutacoes de `tvShows` funcionando
- rate limit, logs, timeout e body limit aplicados
- `npm audit` revisado
