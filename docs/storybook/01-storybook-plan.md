# Storybook 01 - Plano de componentes

## Objetivo

Usar Storybook como catalogo de UI e ambiente de revisao visual.

## Escopo inicial

- button
- input
- textarea
- select
- dialog
- card
- badge
- empty state
- form field wrappers

## Base visual

- `shadcn/ui`
- `Tailwind CSS`
- `Lucide`

## Libs complementares aprovadas

- `motion` para microinteracoes
- `Magic UI` para detalhes pontuais
- `Kibo UI` para componentes utilitarios faltantes

## Regra

Cada lib complementar deve resolver um gap concreto. Nao misturar estéticas incompatíveis.
